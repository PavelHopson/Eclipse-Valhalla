## 🔴 Критические улучшения (нужно сделать)

### 1. XP и уровни не обновляются в UI
**Проблема:** `gamificationService.ts` полностью реализован, но `user.xp` и `user.level` в React-state никогда не обновляются после создания. Пользователь всегда видит "Уровень 1, 0 XP".

**Решение:** При каждом `trackEvent` вызывать `onQuestComplete()` / `onFocusSessionComplete()` из gamificationService и обновлять user state:
```tsx
const updatedUser = { ...user, xp: user.xp + xpReward, level: calculateLevel(user.xp + xpReward) };
setUser(updatedUser);
api.updateUser(user.id, updatedUser);
```

### 2. Toast-уведомления о достижениях
**Проблема:** Когда достижение открывается, пользователь не узнаёт об этом до захода в Зал Славы.

**Решение:** Подписать `onAchievementUnlock()` callback и показывать toast:
```tsx
onAchievementUnlock((achievement) => {
  showToast(`🏆 ${t(`achievements.${achievement.id}`)} — +${achievement.xpReward} XP`);
});
```

### 3. Supabase Cloud Sync
**Проблема:** Всё в localStorage. Потеря данных при очистке браузера, нет синхронизации между устройствами.

**Решение:** `syncEngine.ts` и `syncQueue.ts` уже существуют как заглушки. Нужно:
- Подключить Supabase (URL + Key в .env)
- Реализовать двустороннюю синхронизацию
- Conflict resolution: last-write-wins или merge

## 🟡 Высокий приоритет

### 4. Drag-and-Drop в канбан-доске квестов
**Сейчас:** Карточки перемещаются только кнопками.
**Нужно:** react-dnd или native HTML5 drag для перетаскивания между колонками TODO → IN_PROGRESS → DONE.

### 5. Повторяющиеся квесты — автоматическое создание
**Сейчас:** Поле repeatType есть, но при завершении квеста новый не создаётся.
**Нужно:** В `toggleComplete`, если `repeatType !== 'none'`, создавать новый квест с датой +1 день / +7 дней / +30 дней.

### 6. Push-уведомления
**Сейчас:** Notification API запрашивает разрешение, но не отправляет уведомления о дедлайнах.
**Нужно:** Проверять каждые 5 минут, если квест просрочен — показывать нативное уведомление.

### 7. Экран входа — убрать или упростить
**Сейчас:** Auth.tsx показывает форму логина с паролем, но всё хранится в localStorage.
**Нужно:** Либо сделать полноценную авторизацию через Supabase Auth, либо убрать пароли и оставить только имя.

### 8. Streaming ответов AI
**Сейчас:** Оракул ждёт полный ответ, потом показывает целиком.
**Нужно:** Использовать streaming API (SSE/ReadableStream) для показа текста по мере генерации. Особенно важно для Ollama (32b модель думает 30-60 сек).

## 🟢 Средний приоритет

### 9. Темы оформления
**Сейчас:** Одна тёмная тема с фиксированными цветами.
**Нужно:** 3-5 тем:
- Default (текущая тёмная)
- Blood (красные акценты) — разблокируется наградой
- Gold (золотые) — разблокируется наградой
- Void (чисто чёрная) — разблокируется наградой
- Light (светлая) — для тех кто попросит

### 10. Виджеты на рабочий стол (Electron overlay)
**Сейчас:** Overlay mode есть в коде, но не используется.
**Нужно:** Маленький виджет поверх всех окон: текущий квест + таймер + стрик. Включается из трея.

### 11. Статистика продуктивности
**Сейчас:** Базовая статистика в Dashboard.
**Нужно:** Отдельная вкладка "Аналитика":
- График квестов по дням/неделям
- Тепловая карта активности (как GitHub)
- Среднее время выполнения квестов
- Лучшие дни недели / время суток

### 12. Интеграция с фитнес-трекерами
**Можно:** Импорт данных из Google Fit / Apple Health через CSV/JSON.
**Что даст:** Реальные калории, шаги, ЧСС в статистике тренировок.

### 13. Таймер Помодоро
**Сейчас:** Focus Mode = 25 минут, один квест.
**Нужно:** Настраиваемый Помодоро:
- Рабочий интервал: 25/30/45/60 мин
- Перерыв: 5/10/15 мин
- Длинный перерыв: 15/20/30 мин (каждые 4 цикла)
- Звуковые сигналы

### 14. Привычки (отдельно от квестов)
**Что:** Ежедневные действия без дедлайна — медитация, чтение, вода, сон.
**Как:** Отдельный раздел с чеклистом на каждый день и трекингом процента выполнения.

### 15. Шаблоны квестов
**Что:** Сохранение часто создаваемых квестов как шаблонов.
**Как:** "Сохранить как шаблон" в модалке квеста → быстрое создание из шаблона.
